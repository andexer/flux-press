# Guía para Colaboradores - Flux Press

¡Gracias por tu interés en colaborar con **Flux Press**! Este proyecto busca estandarizar el desarrollo de temas de WordPress bajo los más altos estándares de Laravel y Roots.

Para mantener la calidad y consistencia del código, pedimos a todos los colaboradores que sigan estas directrices.

## Cómo empezar

1. Haz un **Fork** del repositorio.

2. Crea una rama para tu mejora o corrección: `git checkout -b feature/nueva-funcionalidad`.

3. Realiza tus cambios y asegúrate de que el código cumpla con los estándares de estilo.

4. Envía un **Pull Request** detallando los cambios realizados.

## Estándares de Código (Obligatorio)

Para que un Pull Request sea aceptado, el código debe seguir estas reglas estrictas de formato:

- **Tabulaciones:** No uses espacios. Todo el código (PHP, JS, HTML, CSS) debe estar indentado con **tabulaciones (ancho de 4)**.

- **Sin Comentarios:** El código debe ser lo suficientemente limpio y semántico como para explicarse por sí mismo. **No incluyas comentarios** innecesarios o código comentado.

- **Sintaxis Limpia:** Evita saltos de línea excesivos. Buscamos un código elegante, compacto y legible.

- **Tipado:** En PHP, utiliza tipado estricto siempre que sea posible (Return types y Scalar types).

- **Componentes:** Sigue la lógica de componentes de **Flux UI** y **Blade**. Reutiliza antes de crear desde cero.

## Reporte de Errores y Sugerencias

Si encuentras un error o tienes una idea para una nueva funcionalidad:

1. Revisa si ya existe un *Issue* abierto sobre el tema.

2. Si no existe, abre uno nuevo describiendo el problema, los pasos para reproducirlo y el entorno (SO, versión de PHP, etc.).

## Pruebas y Testing

Si tu colaboración incluye lógica compleja en **Livewire 4** o servicios de **Acorn**, te agradecemos incluir pruebas unitarias o de integración que aseguren la estabilidad del tema.
